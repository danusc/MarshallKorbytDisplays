I want you to build an MVP for an internal digital signage “CMS lite” system called Marshall Display Registry.

Goal:
We have Windows public display computers that automatically log in as a kiosk user and launch Google Chrome fullscreen/kiosk to a specific URL. Today this is done with a hardcoded Chrome shortcut in the Startup folder. I want a small internal web app and API where admins can manage which URL each display computer should launch. Each display computer will run a PowerShell endpoint agent that checks in with the API, retrieves its assigned URL, launches Chrome, and reports status.

Hosting target:
- Production hosting will be Azure App Service.
- Production database will be Azure SQL Database.
- Use Azure-native infrastructure where practical.
- Include infrastructure-as-code using Bicep, or Azure Developer CLI-compatible Bicep.
- Include GitHub Actions deployment workflow if practical.
- Local development may use SQL Server LocalDB, Docker SQL Server, or a dev Azure SQL database.
- Do not use SQLite as the main database provider.

Technology choices:
- Use ASP.NET Core, preferably .NET 8 or newer.
- Use Razor Pages or MVC for the admin UI. Keep it simple; do not use React unless absolutely necessary.
- Use Entity Framework Core with the SQL Server provider.
- Use Azure SQL Database in production.
- Use OpenAPI/Swagger for the API.
- Use Application Insights-compatible logging.
- Include clear README setup instructions.
- Include PowerShell scripts for the Windows endpoint agent and installer.

  - Pending: desired URL changed but device has not checked in yet.

  - Error: agent reported an error.

API requirements:
1. POST /api/agent/register
   Request:
   {
     "computerName": "...",
     "serialNumber": "...",
     "macAddress": "...",
     "agentVersion": "..."
   }
   Behavior:
   - If computerName already exists, return existing device config but issue a new device token only if no active token exists.
   - If computerName does not exist, create a display device based on configurable setting AutoEnableNewDevices.
   - Generate a long random device token.
   - Store only a hash of the token.
   - Return the token once.

2. POST /api/agent/checkin
   Auth:
   - Use headers:
     X-Display-ComputerName
     X-Display-Token
   Request:
   {
     "serialNumber": "...",
     "macAddress": "...",
     "agentVersion": "...",
     "currentUrl": "...",
     "chromeRunning": true,
     "lastError": null
   }
   Response:
   {
     "enabled": true,
     "desiredUrl": "...",
     "launchMode": "kiosk",
     "restartChrome": false,
     "pollSeconds": 300,
     "allowedDomains": ["usc.korbyt.com", "marshall.usc.edu", "usc.edu"],
     "commands": [
       {
         "id": 123,
         "commandType": "RestartChrome"
       }
     ]
   }
   Behavior:
   - Validate device token.
   - Update LastSeenUtc, CurrentUrl, ChromeRunning, LastError, AgentVersion.
   - Create DisplayCheckIn record.
   - Return assigned desired URL.
   - Return pending commands.
   - Mark returned commands as PickedUp.

3. POST /api/agent/commands/{id}/complete
   Auth:
   - Same device token headers.
   Request:
   {
     "status": "Completed",
     "resultMessage": "Chrome restarted"
   }
   Behavior:
   - Mark command complete.

4. GET /api/agent/config
   Auth:
   - Same device token headers.
   Behavior:
   - Return current desired config without creating a full check-in record.

PowerShell agent requirements:
Create a folder /agent with:
1. SignageAgent.ps1
2. Install-SignageAgent.ps1
3. Uninstall-SignageAgent.ps1
4. Test-SignageAgent.ps1
5. sample-signage-agent.json

Agent behavior:
- Config file path:
  C:\ProgramData\MarshallDigitalSignage\Agent\signage-agent.json
- Token file path:
  C:\ProgramData\MarshallDigitalSignage\Agent\device-token.txt
- Cache file path:
  C:\ProgramData\MarshallDigitalSignage\Agent\last-known-good.json
- Log folder:
  C:\ProgramData\MarshallDigitalSignage\Logs

SignageAgent.ps1 modes:
- -Mode Register
- -Mode Launch
- -Mode Monitor
- -Mode CheckIn
- -Mode RestartChrome

Agent details:
- Detect computer name using $env:COMPUTERNAME.
- Detect serial number using CIM/WMI.
- Detect primary MAC address.
- Register with the Azure App Service API if no token exists.
- Check in with API using X-Display-ComputerName and X-Display-Token headers.
- Validate returned URL against allowed domains.
- Cache last-known-good URL locally.
- If API is unavailable, use cached URL.
- Launch Chrome from:
  C:\Program Files\Google\Chrome\Application\chrome.exe
- Chrome arguments should include:
  --kiosk or --start-fullscreen based on config
  --autoplay-policy=no-user-gesture-required
  --no-first-run
  --disable-session-crashed-bubble
  --user-data-dir="C:\ProgramData\MarshallDigitalSignage\ChromeProfile"
- The agent should avoid launching duplicate Chrome instances unnecessarily.
- If desired URL changes, close Chrome and relaunch with the new URL.
- If Chrome is closed, relaunch it.
- Write useful logs.

Installer requirements:
Install-SignageAgent.ps1 should:
- Require admin.
- Create C:\ProgramData\MarshallDigitalSignage folders.
- Copy agent files.
- Create or update config file with ApiBaseUrl pointing to the Azure App Service URL.
- Create scheduled task "Marshall Signage - Launch" triggered at logon of axistvuser.
- Create scheduled task "Marshall Signage - Monitor" running every 5 minutes.
- Optionally remove or rename the old Chrome shortcut from:
  C:\Users\axistvuser\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

- Print validation results.

Uninstaller requirements:

- Optionally leave logs/cache behind.
